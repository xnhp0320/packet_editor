# AGENTS.md

This file provides guidance to Qoder (qoder.com) when working with code in this repository.

## Project identity

FlowForge (`ffg`) is a DPDK-native traffic generator driven by a Scapy-like packet description language. It compiles packet programs into high-rate packet streams, with range expressions that expand one template into a set of flows.

## Coding rules

- Use modern C++. Use C++20 aggressively.
- Try best to stay at STL. If you think boost/another library is better, stop and ask.
- Do not link or introduce libpcap. pcap files are written by the DPDK-free `PcapWriter` class.
- Keep live traffic on the DPDK runtime path; keep file output (`-o`) on the DPDK-free pipeline.

## Build

```sh
# Full build including DPDK (default)
cmake -S . -B build
cmake --build build -j2

# DPDK-free build (parser/checker/serializer library + unit tests only)
cmake -S . -B build -DPACKET_BUILD_DPDK=OFF
cmake --build build -j2
```

The full build fetches and builds a bundled DPDK v25.11.2 via `ExternalProject_Add`. DPDK is built with Meson and requires these system packages: `meson`, `pkg-config` (or `pkgconf`), `pyelftools`, `libnuma-dev`, and `git`. A local patch under `patches/dpdk/` prevents libpcap from being linked when `net/pcap` is disabled.

CMake options:

| Option | Default | Effect |
|---|---|---|
| `PACKET_BUILD_DPDK` | `ON` | Build the DPDK runtime and link `ffg` against it |
| `PACKET_BUILD_E2E_TESTS` | `OFF` | Register the Scapy/pytest end-to-end tests |

The `build/compile_commands.json` generated by CMake is symlinked at the repo root for clangd/LSP tooling (see `.clangd`). There is no `.clang-format` or `.clang-tidy` config; follow the existing code style.

## Test

```sh
# All unit tests
ctest --test-dir build --output-on-failure

# A single unit test
ctest --test-dir build -R test_parser --output-on-failure

# End-to-end tests (requires root, Linux TAP, Scapy)
cmake -S . -B build -DPACKET_BUILD_E2E_TESTS=ON
cmake --build build -j2
ctest --test-dir build -R E2ETest --output-on-failure
```

Unit tests use GoogleTest (fetched at build time via `FetchContent`). Each test executable is a standalone binary. E2E tests launch `ffg`, capture packets from the TAP interface `packet_tap0` with Scapy, and validate the emitted packets.

## Architecture

### Processing pipeline

The packet generation pipeline is layered, with DPDK-dependent code cleanly separated from the core:

```
Lexer -> Parser -> Checker -> PacketConstructor -> PacketSerializer -> [Runtime | PcapWriter]
```

1. **Lexer** (`src/lexer.cpp`) — tokenizes the DSL (identifiers, string literals, integers, operators).
2. **Parser** (`src/parser.cpp`) — recursive-descent parser producing an AST (`Program` → `Expression` → `Packet` → `Header` → `Attribute`).
3. **Checker** (`src/checker.cpp`) — validates the AST against the `Registry`: known headers, valid attributes, correct value types (mac, ipv4, ipv6, bit fields, ranges).
4. **PacketConstructor** (`src/packet_constructor.cpp`) — converts validated AST into a structured constructor with resolved defaults, normalized values, inference rules (e.g. `Ether.type=0x0800` when `IP` follows), and bit-level field offsets.
5. **PacketSerializer** (`src/packet_serializer.cpp`) — serializes the constructor into a byte buffer and records `PayloadFieldModifier` objects for each range-valued field. `fixup_packet()` computes IP/TCP/UDP/ICMP checksums or records hardware offload requests.
6. **Runtime** (`src/runtime.cpp`, `src/dpdk_offload.cpp`) — DPDK EAL init, TAP port setup, mbuf pool, packet transmit via `rte_eth_tx_burst()`. Only built when `PACKET_BUILD_DPDK=ON`.
7. **PcapWriter** (`src/pcap_writer.cpp`) — writes pcap files (global header + per-packet records) without requiring libpcap. Used in file mode (`-o`).

### Range expansion

Range expressions like `src="[10.0.0.1-10.0.0.4]"` produce a `PayloadFieldModifier` per range field in the serialized payload. `PacketGenerator` plans a Cartesian product of flow indexes and applies the modifiers per-flow, so only one base payload is serialized; per-flow packets are produced by byte-level modification of the base buffer.

### Registry

The `Registry` class (`src/registry.cpp`) is loaded once at startup with all known protocols (`Ether`, `VLAN`, `IP`, `IPv6`, `TCP`, `UDP`, `ICMP`, `VXLAN`, `Payload`), options headers (`IPOption_*`, `TCPOption_*`), type validators (`mac`, `ipv4`, `ipv6`, `b1`–`b64` with range variants), and inference rules that auto-fill parent header fields based on adjacent child protocols.

### Binary entry point

`apps/dpdk_init.cpp` compiles to `ffg`. It has two modes:

- **Live mode** (no `-o`): reads a `.packet` program file, runs the DPDK tap runtime. Requires `PACKET_BUILD_DPDK=ON`.
- **File mode** (`-o <file>`): accepts a program file or an inline `-e` expression, generates flows, writes pcap. Works without DPDK (`PACKET_BUILD_DPDK=OFF` is sufficient).

### Key types

- `ConstructorValue` (`include/packet/value.hpp`) — variant of `uint64_t`, `MacAddr`, `IPv4`, `IPv6`, `IPv4Range`, `IPv6Range`, and vector-range types.
- `PayloadFieldModifier` — describes a field's bit offset, bit width, byte length, and variant of range values; applied per-flow to mutate the base payload.
- `PacketConstructor` — flat representation of all fields in a packet with their bit offsets, widths, defaults, and user overrides.

## DPDK build details

DPDK is configured with minimal drivers: basic net drivers, no crypto/compress/event/regex/vdpa. The patch at `patches/dpdk/0001-config-do-not-link-libpcap-when-net-pcap-is-disabled.patch` modifies DPDK's `config/meson.build` to skip libpcap detection when `net/pcap` is disabled. Patches are applied via `cmake/ApplyPatchIfNeeded.cmake` — if a patch fails to apply cleanly, the build stops.
